# 实验结果与分析

## 1. 实验设置

### 数据集

| 数据集               | 正负比 | 候选数/步 | 测试集步数 | 说明                                |
| -------------------- | ------ | --------- | ---------- | ----------------------------------- |
| Test（event_group）  | 1:2    | 3         | 68,512     | 默认设置，DQN/DDQN 在此训练         |
| Test1（event_group） | 1:9    | ~10       | 64,408     | 中等难度泛化测试                    |
| Test2（event_group） | 1:19   | ~20       | 62,781     | 更接近真实场景，部分事件候选不足 20 |

### 方法

| 方法             | 类型     | 说明                                                             |
| ---------------- | -------- | ---------------------------------------------------------------- |
| Random           | Baseline | 均匀随机选择                                                     |
| Greedy-Worker    | Baseline | 线性加权打分，权重由 LR 在 worker reward 上学习                  |
| Greedy-Requester | Baseline | 线性加权打分，权重由 LR 在 requester reward 上学习               |
| DQN              | RL       | 深度 Q 网络，MLP [128, 128]，分别用 worker/requester reward 训练 |
| Double-DQN       | RL       | Double DQN，使用目标网络减少 Q 值过估计                         |

### 评估指标

- **Avg Reward**：平均每步累积奖励（核心指标）
- **Hit@1**：推荐命中率（推荐的 == worker 真实选的）
- 评估均在 test 集上进行，使用 event_group 模式

## 3. 分析

### 3.1 DQN 在训练分布内优于 Baseline，但泛化能力有限

在 1:2（训练分布）上，DQN 的 Hit@1（64.6%）显著优于 Greedy（56%）和 Random（33%）。
但在 1:19（分布外）上，DQN（9.0%）反而低于 Greedy（18.6%），Double-DQN（6.1%）甚至接近 Random（5.3%）。

| 数据集 | Random | Greedy-Worker | DQN    | Double-DQN |
| ------ | ------ | ------------- | ------ | ---------- |
| 1:2    | 33.2%  | 56.1%         | 64.6%  | 59.0%      |
| 1:9    | —      | —             | 18.1%  | 12.3%      |
| 1:19   | 5.3%   | 18.6%         | 9.0%   | 6.1%       |

原因分析：
1. **训练分布局限**：DQN/DDQN 仅在 1:2（3 候选）上训练，Q 网络学到的策略高度依赖这一分布。评估时切换到 1:19（~20 候选），候选分布完全不同，Q 值估计失效
2. **Greedy 的鲁棒性**：Greedy 使用线性加权，权重从训练集学到但不依赖候选数量，在分布变化时仍能保持稳定的排序能力
3. **DQN 过拟合训练环境**：MLP [128,128] 在 3 候选场景下能精确区分正负样本，但泛化到 20 候选时，Q 值排序退化

### 3.2 Double-DQN 不如 DQN

在所有数据集上，Double-DQN 的 Hit@1 均低于 DQN：

| 数据集 | DQN Hit@1 | DDQN Hit@1 | 差距    |
| ------ | --------- | ---------- | ------- |
| 1:2    | 64.6%     | 59.0%      | -5.6%   |
| 1:9    | 18.1%     | 12.3%      | -5.8%   |
| 1:19   | 9.0%      | 6.1%       | -2.9%   |

可能原因：
1. **Double-DQN 的解耦更新增加不稳定性**：在小数据集、简单任务上，标准 DQN 的过估计反而提供了一定的探索收益
2. **训练超参数差异**：DDQN 的 target network 更新频率、学习率等可能未充分调优
3. **任务本身简单**：event_group 模式下候选数少（3 个），不需要复杂的 Q 值估计

### 3.3 Greedy Baseline 的 LR 权重学习

传统 Greedy 方法的权重由人工设定（如 awards=2.0, match=1.5），缺乏数据支撑。本项目使用 LR 从训练集数据中学习最优权重：

- LR 在训练集上拟合 P(label=1 | features)，系数即为最优线性权重
- 相比人工设定，LR 权重更客观，但仍是线性模型，无法捕捉非线性关系
- **优势在于泛化**：LR 权重在 1:19 数据集上仍然有效（Hit@1=18.6%），而 DQN 大幅退化

### 3.4 Greedy-Worker 与 Greedy-Requester 差异不大

两种 Greedy 方法在所有实验中 Hit@1 差距仅约 1%，原因：
- 两者使用相同的 10 个特征，LR 学到的权重趋于相似
- 线性模型的表达能力有限，无法充分体现"worker 视角"和"requester 视角"的差异

### 3.5 Reward 模式对比

Worker 和 Requester 模式下各方法的相对排序一致（DQN > Greedy > Random），绝对值差异来自 reward 定义：

```
Worker reward    = label × (1.0 + 0.5×quality + 2.0×winner + 1.0×finalist)
Requester reward = label × (2.0×quality + 1.0×winner)
```

Worker reward 有 base=1.0 保底，正样本 reward 始终 ≥1.0；Requester reward 无保底，quality 低时可能 <0.5。
这导致 Requester 模式的 Avg Reward 整体高于 Worker（因为高 quality 样本 reward 可达 2+），但 Hit@1 一致。

---

## 4. 结论

1. **DQN 在训练分布内最优**：在 1:2 测试集上，DQN Hit@1（64.6%）优于 Greedy（56%）和 Random（33%），证明 RL 方法在匹配场景下有效
2. **DQN 泛化能力有限**：在 1:19 测试集上，DQN（9.0%）反而低于 Greedy（18.6%），说明 RL 模型高度依赖训练时的候选分布
3. **Greedy 泛化最稳定**：LR 学权重的 Greedy 方法在 1:2 和 1:19 上 Hit@1 稳定在 18-56%，不受候选数量变化影响
4. **Double-DQN 未带来增益**：在所有数据集上均不如标准 DQN，可能因为任务简单、过估计反而有利
5. **数据集设计影响评估结论**：仅在 1:2 上评估会高估 DQN 的优势，1:19 更能反映真实推荐场景的难度
