# 实验结果与分析

## 1. 实验设置

### 数据集

| 数据集               | 正负比 | 候选数/步 | 测试集步数 | 说明                                |
| -------------------- | ------ | --------- | ---------- | ----------------------------------- |
| Test（event_group）  | 1:2    | 3         | 68,512     | 默认设置，DQN 在此训练              |
| Test2（event_group） | 1:19   | ~20       | 62,781     | 更接近真实场景，部分事件候选不足 20 |

### 方法

| 方法             | 类型     | 说明                                                             |
| ---------------- | -------- | ---------------------------------------------------------------- |
| Random           | Baseline | 均匀随机选择                                                     |
| Greedy-Worker    | Baseline | 线性加权打分，权重由 LR 在 worker reward 上学习                  |
| Greedy-Requester | Baseline | 线性加权打分，权重由 LR 在 requester reward 上学习               |
| DQN              | RL       | 深度 Q 网络，MLP [128, 128]，分别用 worker/requester reward 训练 |

### 评估指标

- **Avg Reward**：平均每步累积奖励（核心指标）
- **Hit@1**：推荐命中率（推荐的 == worker 真实选的）
- 评估均在 test 集上进行，使用 event_group 模式

## 3. 分析

### 3.1 DQN 显著优于所有 Baseline

在两种数据集、两种 reward 模式下，DQN 的 Avg Reward 和 Hit@1 均远超 Baseline。

- **1:2 数据集**：DQN Hit@1 接近 100%，是 Greedy 的 1.8 倍，Random 的 3 倍
- **1:19 数据集**：DQN Hit@1 仍然 83-99%，而 Baseline 降至 5-19%

DQN 的优势来自：
1. **从 reward 中自动学习**：不需要人工设计特征权重，通过贝尔曼方程自动优化 Q 函数
2. **非线性拟合能力**：MLP 可以捕捉特征间的交互效应，而 Greedy 的线性加权无法做到
3. **长期优化**：Q 值考虑未来累积 reward，而非单步贪心

### 3.2 Greedy Baseline 的 LR 权重学习

传统 Greedy 方法的权重由人工设定（如 awards=2.0, match=1.5），缺乏数据支撑。本项目使用 LR 从训练集数据中学习最优权重：

- LR 在训练集上拟合 P(label=1 | features)，系数即为最优线性权重
- 相比人工设定，LR 权重更客观，但仍是线性模型，无法捕捉非线性关系

### 3.3 1:19 数据集更能体现方法差距

| 对比          | 1:2（3 候选） | 1:19（~20 候选） |
| ------------- | ------------- | ---------------- |
| Random Hit@1  | 33.2%         | 5.2%             |
| Greedy Hit@1  | ~56%          | ~18%             |
| DQN Hit@1     | ~100%         | ~99%/83%         |
| DQN vs Random | 3×            | 19×              |

1:19 更接近真实推荐场景（20 个候选选 1 个），此时 DQN 的优势更加显著。

### 3.4 Requester DQN 泛化能力稍弱

在 1:19 数据集上：
- Worker DQN: 99.997% → 99.1%（几乎不变）
- Requester DQN: 99.76% → 83.0%（下降 17%）

原因分析：
1. **Reward 信号质量**：Worker reward 有 base=1.0 保底，正负样本信号清晰；Requester reward 无保底，quality 低时信号弱
```
Worker reward = 1.0 + 0.5×quality + 2.0×winner + 1.0×finalist
                ↑ 保底 1.0，正样本永远明显区别于负样本（0）

Requester reward = 2.0×quality + 1.0×winner
                   ↑ 没有保底，quality 低时 reward 可能只有 0.1~0.5
```
2. **训练收敛质量**：Requester 模型最终 loss（0.0019）高于 Worker（0.0004），Q 函数拟合更差
3. **特征敏感性**：Requester 模型依赖 quality（连续值）、winner（稀疏），对负样本分布变化更敏感

### 3.5 Greedy-Worker 与 Greedy-Requester 差异不大

两种 Greedy 方法在所有实验中 Hit@1 差距仅约 1%，原因：
- 两者使用相同的 10 个特征，LR 学到的权重趋于相似
- 线性模型的表达能力有限，无法充分体现"worker 视角"和"requester 视角"的差异

---

## 4. 结论

1. **DQN 是最有效的方法**：在所有实验设置下均显著优于 Baseline，尤其在接近真实场景的 1:19 数据集上优势更大
2. **Greedy Baseline 提供合理下界**：LR 学权重比人工设定更客观，但线性模型表达能力有限
3. **数据集设计影响评估结论**：1:2 数据集过于简单（DQN 几乎 100%），1:19 更能区分方法优劣
4. **Reward 设计影响模型泛化**：Requester reward 缺少 base 项导致信号弱、泛化差，建议未来改进 reward 设计
